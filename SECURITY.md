# Security Policy

## Scope

This repository contains public GitHub Pages / portfolio site content. It should not contain secrets, private credentials, internal hostnames, or sensitive operational details.

## Reporting a Vulnerability

Please do not open public issues for security concerns.

Report privately through GitHub private vulnerability reporting if enabled, or contact the maintainer directly.

Please include:

- affected page or file
- description of the issue
- potential impact
- suggested fix, if available

## Security Expectations

- Do not commit API keys, tokens, private keys, or `.env` files.
- Do not publish private infrastructure details.
- Keep links, badges, and project references accurate.
- Prefer public-safe contact methods and GitHub noreply addresses.
